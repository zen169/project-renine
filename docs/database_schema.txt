============================================================
DATABASE INSTANCES
============================================================
All databases are locally stored SQLite files with an accompanying
ChromaDB directory for semantic embeddings.

- data/history.db     — Conversation history (Layer 2)
- data/mind.db        — Structured facts (Layer 3)
- data/personality.db  — People profiles (Layer 4)
- data/chroma/        — Vector database containing embeddings

============================================================
TABLE SCHEMA DETAILS
============================================================

------------------------------------------------------------
history.db -> conversations
------------------------------------------------------------
Stores summaries of past conversation sessions with raw chat turns.
- id (INTEGER): Primary Key, Autoincrement
- date (DATE): Date of the conversation session (NOT NULL)
- summary (VARCHAR): Text summary of the session (NOT NULL)
- raw_turns (JSON): Raw list of chat turns (role, content) (NOT NULL)
- created_at (DATETIME): Record creation timestamp (NOT NULL)

------------------------------------------------------------
mind.db -> mind_records
------------------------------------------------------------
Stores key-value configurations and facts classified by namespaces.
- id (INTEGER): Primary Key, Autoincrement
- namespace (VARCHAR): Name of the context namespace (NOT NULL, Index)
- key (VARCHAR): Unique key identifier within the namespace (NOT NULL)
- value (JSON): Associated structured data (NOT NULL)
- summary (VARCHAR): Short text description used for vector embedding (NOT NULL)
- created_at (DATETIME): Record creation timestamp (NOT NULL)
- updated_at (DATETIME): Record last updated timestamp (NOT NULL)
* Unique Constraint: UniqueIndex on (namespace, key)

------------------------------------------------------------
mind.db -> inventory
------------------------------------------------------------
Stores items in the household inventory (food, ingredients, supplies).
- id (INTEGER): Primary Key, Autoincrement
- name (VARCHAR): Name of the item (NOT NULL, Unique Index)
- category (VARCHAR): Category (food/ingredient/supply) (NOT NULL, Index)
- quantity (FLOAT): Current quantity in stock (NOT NULL)
- unit (VARCHAR): Unit of measure (NOT NULL)
- threshold (FLOAT): Minimum quantity threshold for alerts/shopping (NOT NULL)
- location (VARCHAR): Where it is stored (NOT NULL)
- expiration_date (DATETIME): Expiration timestamp (NULLABLE)
- created_at (DATETIME): Record creation timestamp (NOT NULL)
- updated_at (DATETIME): Record last updated timestamp (NOT NULL)

------------------------------------------------------------
mind.db -> pets
------------------------------------------------------------
Stores pet profiles and schedules.
- id (INTEGER): Primary Key, Autoincrement
- name (VARCHAR): Name of the pet (NOT NULL, Unique Index)
- species (VARCHAR): Species (dog/cat/etc) (NOT NULL)
- breed (VARCHAR): Breed (NULLABLE)
- age (FLOAT): Age in years (NULLABLE)
- birthday (VARCHAR): Birthday string (NULLABLE)
- weight (FLOAT): Weight in kg (NULLABLE)
- feeding_schedule (JSON): Scheduled feeding times (NOT NULL)
- medical_conditions (JSON): List of medical conditions (NOT NULL)
- medications (JSON): List of active medications and schedules (NOT NULL)
- last_fed (DATETIME): Timestamp when last fed (NULLABLE)
- created_at (DATETIME): Record creation timestamp (NOT NULL)
- updated_at (DATETIME): Record last updated timestamp (NOT NULL)

------------------------------------------------------------
mind.db -> house
------------------------------------------------------------
Stores rooms, appliances, and furniture tracking details.
- id (INTEGER): Primary Key, Autoincrement
- name (VARCHAR): Name or description of the item (NOT NULL, Unique Index)
- item_type (VARCHAR): Type of item (room/appliance/furniture) (NOT NULL, Index)
- room (VARCHAR): Room where item is located (NOT NULL, Index)
- status (VARCHAR): Status (functional/maintenance/broken) (NOT NULL)
- details (JSON): Custom metadata dict (brand, purchase date, etc.) (NOT NULL)
- created_at (DATETIME): Record creation timestamp (NOT NULL)
- updated_at (DATETIME): Record last updated timestamp (NOT NULL)

------------------------------------------------------------
personality.db -> people
------------------------------------------------------------
Stores structured user profiles and contact/relationship notes.
- id (INTEGER): Primary Key, Autoincrement
- name (VARCHAR): Unique name of the person (NOT NULL, Unique)
- relationship (VARCHAR): Relationship type (NULLABLE)
- age (INTEGER): Age of the person (NULLABLE)
- birthday (VARCHAR): Birthday string (NULLABLE)
- food_preferences (JSON): List of food preferences (NULLABLE)
- hobbies (JSON): List of hobbies (NULLABLE)
- personality_traits (JSON): List of traits (NULLABLE)
- goals (JSON): List of goals (NULLABLE)
- habits (JSON): List of habits (NULLABLE)
- notes (VARCHAR): Additional notes on personality (NULLABLE)
- created_at (DATETIME): Record creation timestamp (NOT NULL)
- updated_at (DATETIME): Record last updated timestamp (NOT NULL)

============================================================
VECTOR SCHEMA DETAILS (ChromaDB)
============================================================
- mind_[namespace]: Collections mapped to Layer 3 mind records. Embedded using document=summary, metadata={namespace, key}.
- personality: Collection mapped to Layer 4 profile notes. Embedded using document=consolidation_string, metadata={name}.
