ARCHITECTURE & DESIGN DECISIONS
============================================================
1. Dual-Engine Storage Pattern:
   SQLite is used for structured, relational state representation 
   (relational source of truth), while ChromaDB acts as an indexing 
   layer for semantic similarity search. This separates transactional 
   guarantees from semantic lookup concerns.
   
2. Unified Memory API (MemoryManager):
   Centralizes access to all memory layers. Prevents direct agent 
   access to raw SQLAlchemy sessions or vector client objects, 
   ensuring uniform formatting, security sanitization, and locking.

3. Background Time-to-Live Expiration:
   An APScheduler background daemon executes periodic cleanups on 
   Layer 2 conversation records. Hardcoded to a 48-hour TTL threshold 
   as specified in settings.yaml.

4. Context Injection Pipeline:
   When MainBrainAgent processes incoming user inputs, it calls 
   MemoryAgent.process() synchronously. MemoryAgent performs parallel 
   retrievals across:
     - Recent conversation turnovers (Layer 2)
     - Relevant local facts (Layer 3)
     - Relevant people profiles (Layer 4)
   The results are formatted as system message context, prepended 
   to the message history sent to Ollama (Qwen3 8B).
